Flyswatter 3000



A videogame project I made myself during 2016. View samples of the source code on GitHub. Download and play the full game from this Dropbox link.

It is an arcade-style top-down shooter, where the player must survive against waves of enemies (giant mutant flies, that have taken over the world). Some of the features include randomly generating maps, increasing difficulty, online leaderboards, and systems for weapon changing, ammo, and powerup collection.

The game was made using Unity Engine, and scripted in C#. Graphics were drawn using GIMP. Sound effects were generated at www.bfxr.net, and leaderboards hosting is provided by dreamlo.com

This project has taught me a lot about the different aspects of game development. Working alone, even such a modest project requires a surprising amount of work, and some planned features will eventually be left out. Naturally, I also gained programming experience in C#, and managed to come up with clever solutions to some problems, such as randomized map generation shown in the sample scripts.


Phrase sorting in C

An assignment for 'Data Structures and Algorithms' course. View the full source code here.

The program can take very large text files as input, that contain phrases and associated phrases separated by a colon. The phrases are initially stored into a hash table, that makes finding and handling duplicate phrases much faster than a regular array. The phrases are then sorted according to their association count (If even more efficiency was required, heap sort could be used instead), and finally the program prints the 100 most common phrases. If the user wants, the top phrases' associations are printed as well.

Having learned the basics of programming in C, this coursework naturally reinforced my C skills even further. It also taught me a lot about the importance of efficiency, regarding the choice of algorithms and data structures used. Although handling strings in C is somewhat primitive, sometimes it is necessary to use a lower level language that has support for memory management


Simple Hangman Game

A course assignment for 'Object Oriented Programming'. View the full source code here.

It is a simple command-line implementation of the classic hangman game. A file of possible words to guess is read and stored into an object that contains a list. Another object keeps track of the game's situation.

I did many small programming exercises during the object-oriented programming course. I chose to show off this one in particular, because it showed me the what objects can be used for in a practical context.
